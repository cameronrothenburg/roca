ROCA LANGUAGE MANUAL
====================

Roca is a contractual language that compiles to JavaScript.
Every function has proof tests. Every error is handled by crash blocks.
Function bodies are pure happy path — no error variables, no if err checks.

HAPPY PATH PHILOSOPHY
---------------------
Function bodies contain ONLY the success case. Errors are separate.

  Happy path:  The function body — what happens when everything works.
  Bad path:    The crash block — what happens when something fails.

This means:
  - No null returns. If something might not exist, use -> Type, err
    with a named error like err.not_found. The caller handles it
    in their crash block, not with null checks.
  - No error variables in the body. Crash blocks intercept errors
    before they reach your code. The body never sees them.
  - No if err checks. The crash block decides what happens: halt
    (propagate), fallback (default value), retry, or panic.

Example — WRONG (null return):

  pub fn find(id: String) -> User | null {    // ✗ nullable return
      ...
  }

Example — RIGHT (error return):

  pub fn find(id: String) -> User, err {      // ✓ happy path
      err not_found = "user not found"
      ...
  }

The caller's crash block decides what to do:

  const user = find(id)
  crash { find -> fallback(fn(e) -> default_user) }

COMMANDS
--------
  roca init <name>       Create a new project
  roca check [path]      Parse + lint + type check
  roca build [path]      Check → build JS → run proof tests
  roca test [path]       Build + test, then clean output
  roca run [path]        Build + execute via bun
  roca lsp               Start language server
  roca man               This manual
  roca --version         Print version

All commands read roca.toml for configuration when no path given.

FUNCTIONS
---------
Every function has three sections: logic, crash, test.

  pub fn greet(name: String) -> String {
      const trimmed = name.trim()
      return "Hello " + trimmed
      crash { name.trim -> skip }
      test {
          self("cam") == "Hello cam"
          self(" cam ") == "Hello cam"
      }
  }

- fn = private, pub fn = exported
- Every function MUST have a test {} block
- Every call MUST appear in the crash {} block
- Functions that can fail declare -> Type, err

TYPES
-----
Primitives: String, Number, Bool
Optional:   Optional<String> — field may be absent (use on struct fields)
Generic:    Array<T>, Map<V>
Special:    Ok (void return)

VARIABLES
---------
  const x = 5               immutable
  let x = 5                 mutable, can reassign
  self.field = value         mutate struct fields in methods

CONTRACTS
---------
Contracts declare what — signatures, errors, mocks. No implementation.

  contract HttpClient {
      get(url: String) -> String, err {
          err timeout = "request timed out"
          err not_found = "404 not found"
      }
      mock {
          get -> "mock response"
      }
  }

Generic contracts:

  contract Array<T> {
      push(item: T) -> Number
      pop() -> T
      map(callback: T) -> Array
      filter(callback: T) -> Array<T>
  }

  contract Logger<T: Loggable> {
      add(item: T) -> Number
  }

STRUCTS
-------
Structs implement contracts. Two blocks: first {} is the contract,
second {} is the implementation.

  pub struct Email {
      value: String
      validate(raw: String) -> Email, err {
          err missing = "email is required"
          err invalid = "email format is not valid"
      }
  }{
      fn validate(raw: String) -> Email, err {
          if raw == "" { return err.missing }
          if !raw.includes("@") { return err.invalid }
          return Email { value: raw }
          crash { raw.includes -> skip }
          test {
              self("a@b.com") is Ok
              self("") is err.missing
              self("bad") is err.invalid
          }
      }
  }

Field constraints (must include a default):

  pub struct User {
      name: String { min: 1, max: 64, default: "unknown" }
      email: String { contains: "@", min: 3, default: "none@none.com" }
      age: Number { min: 0, max: 150, default: 0 }
  }{}

Available: min, max, minLen, maxLen, contains, pattern, default.

SATISFIES
---------
Links a struct to a contract. One block per contract.

  Email satisfies Loggable {
      fn to_log() -> String {
          return self.value
          test { self() == "test@example.com" }
      }
  }

ERRORS
------
Errors are contracts with name + message:

  err missing = "value is required"

Returns { name: "missing", message: "value is required" }.
Override the message at return site:

  return err.missing("name field cannot be blank")

Error returns include a zero value: ("", err) not (null, err).
Tests match on .name: self("") is err.missing

CRASH BLOCKS
------------
Crash blocks declare how each call's errors are handled.
Strategies chain with |>:

  crash {
      db.save -> log |> retry(3, 1000) |> halt         // log, retry, propagate
      config.load -> panic                             // crash process
      fetch -> fallback("default")                     // use static value
      db.query -> fallback(fn(e) -> e.message)         // use error value
      http.get {                                       // per-error handling
          err.timeout -> log |> retry(3, 1000) |> halt
          err.not_found -> fallback("empty")
          default -> log |> halt
      }
  }

Only calls to functions that return errors (-> Type, err) need crash entries.
Stdlib methods (trim, push, etc.) are safe and don't need crash entries.

Strategies:
  halt             Unwrap [value, err] and propagate error to caller
  log              Log the error, continue chain
  retry(n, ms)     Retry n times with ms delay
  fallback(value)  Use a default value
  fallback(fn(e))  Closure receives error with .name and .message
  panic            console.error + process.exit(1)

Chains ending in halt → caller must declare those errors.
Chains ending in fallback/panic → error is consumed.

TEST BLOCKS
-----------
  test {
      self(1, 2) == 3              assert equality
      self("a@b.com") is Ok        assert no error
      self("") is err.missing      assert specific error
  }

- self() calls the function being tested
- Every error return must be tested
- Every function must have at least one success case
- Empty test {} is allowed for instance methods
- Async functions are automatically awaited

CONTROL FLOW
-------------
  if condition { } else { }
  for item in items { }
  while condition { break / continue }
  match value {
      200 => "ok"
      404 => err.not_found
      _ => "unknown"
  }

CLOSURES
--------
  items.map(fn(x) -> x * 2)
  items.filter(fn(x) -> x > 5)
  const double = fn(x) -> x * 2

ASYNC (WAIT)
------------
  const data = wait http.get("/api")
  let a, b = waitAll { call1() call2() }
  let fastest = waitFirst { call1() call2() }

Functions with wait auto-become async. No async keyword needed.
Errors from wait calls are handled by crash blocks.

OPTIONAL FIELDS
---------------
Use Optional<T> for struct fields that may be absent:

  pub struct Profile {
      name: String
      bio: Optional<String>
  }{}

  // Access with methods:
  if profile.bio.isPresent() { ... }
  const text = profile.bio.unwrapOr("No bio")

Do NOT use Type | null — use Optional<T> for data, -> Type, err for functions.

Type casts:
  let n, err = Number("42")   safe cast, returns err on invalid
  let s, err = String(42)     safe cast

ENUMS
-----
  enum Status { active = "active", suspended = "suspended" }
  enum HttpCode { ok = 200, not_found = 404 }

Compiles to: const Status = { active: "active", suspended: "suspended" };

IMPORTS
-------
  import { Email } from "./email.roca"
  import { User } from "./user.roca"

Compiles to: import { Email } from "./email.js";

EXTERN DECLARATIONS
-------------------
Declare JS runtime types and functions. No JS emitted.

  extern contract Database {
      query(sql: String) -> String, err {
          err query_failed = "database query failed"
      }
      mock {
          query -> "[]"
      }
  }

  extern fn log(msg: String) -> Ok

Mock blocks provide test doubles for proof tests.

EXTERN DEPENDENCIES
-------------------
Pass external dependencies as explicit parameters — not bundled in an env bag:

  extern contract Database {
      query(sql: String) -> String, err {
          err query_failed = "database query failed"
      }
      mock { query -> "[]" }
  }

  pub fn get_users(db: Database) -> String, err {
      err query_failed = "database query failed"
      const data = wait db.query("SELECT * FROM users")
      return data
      crash {
          db.query -> halt
      }
      test {
          self(__mock_Database) is Ok
      }
  }

The function signature declares exactly what it needs. No env struct.

JS side wires the real implementation:

  import { get_users } from "my-roca-lib";

  const db = {
      query: (sql) => {
          const rows = await cfEnv.DB.prepare(sql).all();
          return { value: JSON.stringify(rows.results), err: null };
      }
  };

  const { value, err } = await get_users(db);

JSLIB MODE
----------
In roca.toml:

  [build]
  mode = "jslib"

roca build produces out/package.json and runs npm install.
Your JS project can then: import { handler } from "my-lib"

STDLIB
------
String:  trim, trimStart, trimEnd, toUpperCase, toLowerCase, replace,
         slice, repeat, includes, startsWith, endsWith, indexOf, split,
         charAt, toString, to_log
Number:  toString, toFixed, to_log
Bool:    toString, to_log
Array:   length, includes, indexOf, join, reverse, push, pop, concat,
         map, filter, find, slice, forEach, some, every, flat, reduce
Map:     get, set, has, delete, keys, values, size
Bytes:   byteLength, at, slice, toString, to_log, toHex, toBase64, toArray
Buffer:  write, writeString, writeByte, toBytes, toString, byteLength, clear
Optional: isPresent, unwrap, unwrapOr

Loggable: contract requiring to_log() -> String.
String, Number, Bool, Bytes all satisfy Loggable.
log(), error(), warn() require Loggable arguments.

COMPILER RULES
--------------
The compiler enforces:
  missing-test         Every function needs a test block
  missing-crash        Every call needs a crash handler
  unhandled-call       Crash block missing a handler for a call
  untested-error       An error return path has no test case
  no-success-test      Error tests without a success case
  const-reassign       Cannot reassign a const variable
  unknown-method       Method doesn't exist on the type
  type-mismatch        Comparing different types
  struct-comparison    Cannot compare structs directly
  nullable-access      Calling method on nullable without null check
  not-loggable         log/error/warn arg doesn't have to_log()
  generic-mismatch     Wrong type for generic parameter
  constraint-violation Type doesn't satisfy generic constraint
  return-type-mismatch Returning wrong type from function
  return-null          Returning null from non-nullable function
  return-err-not-declared  Returning error from non-err function
  arg-type-mismatch    Wrong argument type at call site
  field-type-mismatch  Wrong field type in struct literal
  unknown-field        self.field doesn't exist on struct
  type-annotation-mismatch  let x: Number = "hello"
  unhandled-error      Error propagates via halt but not declared
  err-in-body          Use crash block, not let val, err = call()
  manual-err-check     Use crash block, not if err { ... }
  missing-default      Constrained field has no default value
  invalid-constraint   Bad field constraint (min > max, etc.)
  duplicate-err        Duplicate error name in contract
  missing-impl         Struct contract method not implemented
  sig-mismatch         Implementation doesn't match signature
  satisfies-mismatch   Satisfies method doesn't match contract
  nullable-type        Type | null used — require the value or use a default
  nullable-return      Return Type | null — use -> Type, err instead
  crash-on-safe        Crash entry on function that doesn't return errors
  panic-warning        panic will crash the process — use halt or fallback
  empty-struct         Struct has no methods — use a contract instead
  invalid-mock-ref     __mock_X used but X has no mock block
  private-method       Calling non-pub struct method from outside

CONFIGURATION (roca.toml)
-------------------------
  [project]
  name = "my-app"
  version = "0.1.0"

  [build]
  src = "src/"
  out = "out/"
  mode = "jslib"         # optional: auto npm install
  tracking = false       # optional: disable compilation logs

OBSERVABILITY
-------------
All compilation events logged to ~/.roca/logs/roca.jsonl:
  parse_error    — file, message, source code
  check_errors   — file, errors with code/message/context, source
  test_result    — file, passed/failed count, output
  build_success  — file, output path
  build_failed   — file, reason

Disable with tracking = false in roca.toml.
